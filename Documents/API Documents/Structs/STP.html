  • configure(applicationToken:endPoint:completion:terminalErrorHandler:)

    コンフィグレーション

    • アプリケーション起動時に一度だけ呼んでください
    • applicationTokenが間違っている場合はエラーが返り、STPRES Payments SDK は動作しません

    Declaration

    Swift

    static func configure(
    applicationToken: String,
    endPoint: STPSDKEndpoint.Endpoint = .product,
    completion: @escaping (Result<Void, Error>) -> Void,
    terminalErrorHandler: ((TerminalError) -> Void)? = nil
)

    Parameters

    applicationToken

    登録したBundle ID毎に発行されたApplication Token
    endPoint

    接続先指定 ※一部の許可されたアプリケーション以外は全てこの設定は無効となり、.product で動作します。
    completion

    本メソッドの実行結果が非同期で返ります
    terminalErrorHandler

    決済端末への接続結果が非同期で返ります
  • setup(_:)

    設定を変更します。 このメソッドで変更した値はアプリケーション起動中はSDKで保持されます。

    • このメソッドを呼ばない場合は初期値で動作します。

    Declaration

    Swift

    static func setup(_ settings: Settings)

    Parameters

    settings

    設定項目

    • isPrintReceiptEnabled: レシート印刷を有効 初期値 false
    • isSendReceiptMailEnabled: レシートメール送信を有効 初期値 false
    • mode: SDK 操作モード 初期値 .staffOperation
